# Social Features

Phase 1A creates architecture only. Flim should not become a social feed-first app.

The social layer exists to make movie playlists easier to share, save, clone, and collaborate on.

## Future Capabilities

- Follow playlists.
- Save playlists.
- Clone playlists.
- Share playlists.
- Collaborate on playlists.
- See who recommended movies.
- See playlist popularity.
- See follower counts.

## Boundaries

- No social feeds in Phase 1A.
- No authentication in Phase 1A.
- No notifications in Phase 1A.
- No comments or discussions in Phase 1A.
- No popularity calculations in Phase 1A.

## Product Principle

Social features should support playlist utility, not distract from the core poster-first browsing experience.
